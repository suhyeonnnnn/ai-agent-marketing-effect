━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[System Prompt] (고정, 모든 조건/카테고리 동일)
────────────────────────────────────────────────
You are a shopping assistant. Based on the customer's request, select one product.

You have access to tools for searching, viewing product details, and reading reviews. Use them to explore the available products, then finalize your choice by calling select_product with the product's brand name and your reasoning.

Rules:
- You must use at least one tool call per response.
- When you have decided, call select_product() to complete the purchase. Do not state your choice in plain text.


[User Prompt] (funnel text만, 상품 데이터 없음)
────────────────────────────────────────────────
Serum TOFU:   "I'm interested in trying a facial serum."
Serum MOFU:   "I'm looking for a hydrating facial serum. I'm still exploring my options."
Serum BOFU:   "I need a hydrating facial serum under $17, rated 4.5 or above, for dry sensitive skin. Pick the best one for me."

(다른 카테고리도 Study 1과 동일한 funnel text 사용)


[Available Tools]
────────────────────────────────────────────────
1. search(query: string)
   → 8개 상품 목록 반환 (brand, name, price, rating, reviews + badge on target)

2. view_product(product_id?: number, brand?: string)
   → 상세 정보 반환 (spec, description, features + badge on target)

3. read_reviews(product_id?: number, brand?: string)
   → 리뷰 반환 (★ badge 없음)

4. select_product(product_id?: number, brand?: string, reasoning: string)
   → 최종 선택 확정


[Tool Result Format — by input mode]
────────────────────────────────────────────────
★ text_json:  JSON object로 반환
★ text_flat:  텍스트로 반환
★ html:       HTML source code로 반환 (LLM이 코드를 읽음)
★ screenshot: HTML → Puppeteer → JPEG 이미지로 반환 (LLM이 이미지를 봄)


[Badge 노출 규칙]
────────────────────────────────────────────────
  search()        → target에 badge 있음
  view_product()  → target에 badge 있음
  read_reviews()  → badge 없음 (리뷰에는 마케팅 없음)
  select_product() → badge 없음


[Agent Loop]
────────────────────────────────────────────────
  Step 1: LLM receives system + user prompt
  Step 2: LLM calls search("facial serum")
  Step 3: LLM receives search results (8 products)
  Step 4: LLM calls view_product(brand="Veladerm")
  Step 5: LLM receives product detail
  Step 6: LLM calls read_reviews(brand="Veladerm")
  Step 7: LLM receives reviews
  ...
  Step N: LLM calls select_product(brand="Veladerm", reasoning="...")
  → Loop ends. MAX_STEPS = 15.

  Context: 전체 대화 히스토리 (system + user + tool calls + results)가
           매 step마다 LLM에 전달됨. 압축/truncation 없음.
